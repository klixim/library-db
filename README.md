# Управление Библиотекой

Проект представляет собой систему управления библиотекой (Library DB), предназначенную для автоматизации учета книжного фонда, управления читателями и отслеживания выдачи и возврата литературы.

## Основной функционал

- **Администрирование:** добавление, удаление и редактирование информации о книгах, управление пользователями, просмотр истории выдачи.
- **Для читателей:** просмотр доступного каталога книг, получение информации о своих взятых книгах.
- **Безопасность:** защита маршрутов API с помощью JWT (Bearer-токен).

## Авторизация (Ролевая модель)
В системе реализовано 2 уровня доступа через модальное окно. Для защиты эндпоинтов используется Bearer-токен.

### Данные для входа:

**Администратор (Библиотекарь):**
- **Логин:** `admin`
- **Пароль:** `admin123`

**Читатель:**
- **Логин:** `reader`
- **Пароль:** `reader123`

## Структура базы данных

Основные сущности системы:
- `Users` — данные пользователей и их роли (администратор, читатель).
- `Books` — каталог литературы с указанием автора, жанра и количества доступных экземпляров.
- `Loans` — учет выданных и возвращенных книг (связующая таблица между читателем и книгой).

## Основные API Эндпоинты

- `POST /api/auth/login` — аутентификация и получение Bearer-токена.
- `GET /api/books` — получение списка всех книг.
- `POST /api/books` — добавление новой книги (Доступ: **Администратор**).
- `POST /api/loans` — оформление выдачи книги читателю.
